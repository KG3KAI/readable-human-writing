# 文档结构模板

把模板当作起点，不要机械套用。先判断读者要做什么，再选择最小的有效结构。

## 选择规则

| 读者任务 | 默认结构 | 适合场景 |
|---|---|---|
| 快速得到答案 | 结论 → 关键理由 → 限制/下一步 | 问答、聊天、简短说明 |
| 做一个选择 | 推荐 → 判断标准 → 取舍 → 行动 | 方案评估、采购、决策备忘录 |
| 理解为什么现在要处理 | SCQA → 答案 → 证据 → 行动 | 管理层汇报、变更申请 |
| 形成一页战略 | 事实 → 问题 → 想法 → 方案 → 指标/风险 | 战略、产品规划、业务方案 |
| 解释项目怎么推进 | 现状 → 问题 → 目标 → 方案 → 资源 → 风险 → 下一步 | 项目立项、实施方案 |
| 复盘结果 | 目标 → 结果 → 偏差 → 原因 → 经验 → 行动 | 周报、复盘、故障总结 |
| 沉淀知识或研究 | 核心结论 → 方法/证据 → 分析 → 局限 → 启示 | 研究报告、技术调研 |
| 教读者完成任务 | 目标 → 前置条件 → 步骤 → 验证 → 常见问题 | 教程、操作手册 |
| 将问答沉淀为文档 | 材料提取 → 口径确认 → 核心结论 → 主题重组 → 未决问题/行动 | 多轮问答、访谈、咨询记录 |
| 改善已有文字 | 保留事实与意图 → 修复结构 → 润色语言 | 润色、改写、编辑 |

## 复合文档

当一份文档同时包含教程、复盘、方案、协作规范等内容时，先选择一个主框架决定全文阅读路径，再在相关章节内使用局部框架。不要把多个完整模板平铺在同一层级。

```text
主框架：决定全文为什么这样展开
局部框架：解决某一章节内部怎么讲清楚

示例：
主框架：经验复盘（目标 → 结果 → 偏差 → 原因 → 经验 → 行动）
局部框架：
- 环境搭建章节使用 How-to
- Bug 章节使用现象 → 根因 → 修复 → 验证
- Git 章节使用原则 → 操作 → 风险 → 验收
```

相邻章节应存在至少一种明确关系：时序、因果、递进、对比、拆解、证据或行动承接。若无法说明关系，优先调整顺序、合并章节或删除无关内容，不要只添加“接下来”“此外”等过渡词。

## 问答转文档

不要按照对话时间顺序拼接。先把对话内容归入以下材料池：

```text
已确认事实：有明确来源或双方已确认的内容
实际观察：在执行、走查或讨论中直接看到的现象
核心结论：对事实的主要解释或判断
原因推断：基于事实和观察得出的解释，尚需与证据边界一致
验证结果：通过测试、运行或验收明确确认的结果
建议/方案：准备采取的做法
分歧：仍存在不同意见的判断
假设：尚未验证但影响结论的前提
行动项：负责人、动作、截止时间（如已知）
未决问题：必须确认或补充的信息
```

生成前，如果读者、用途、篇幅或证据边界会影响结果且尚未明确，先提出一个带默认值的合并确认。用户确认或授权自行决定后，再根据用途选择本文件中的正式结构。

默认规则：

- 不把提问过程、寒暄、重复确认和已经被推翻的中间答案写进正文。
- 不把模型推断伪装成用户已经确认的事实。
- 对关键缺口使用 `待确认`；不要用流畅措辞掩盖信息缺失。
- 若需要保留讨论过程，把它放入“决策依据”或附录，而不是正文主线。

## SCQA

- **S（背景）**：读者已知的稳定事实。
- **C（冲突）**：变化、约束、风险或矛盾。
- **Q（问题）**：读者需要回答的决策问题。
- **A（答案）**：一句可执行的核心结论。

常用骨架：

```text
背景：……
变化/冲突：……
需要回答的问题：……
结论：建议……，因为……
```

## 一页战略

```text
事实：现在发生了什么？
问题：真正阻碍目标的关键矛盾是什么？
想法：应当采取什么核心判断？
方案：具体做什么、不做什么？
指标：如何知道方案有效？
风险：什么条件会让方案失效？
```

## 论证树

先写树，再写段落：

```text
核心结论：……
├─ 理由一：……
│  ├─ 证据：……
│  └─ 影响：……
├─ 理由二：……
│  ├─ 证据：……
│  └─ 取舍：……
└─ 边界/风险：……
```

每个分支都应能回答：它支持上层结论吗？和兄弟分支重复吗？是否缺少决定性证据？

## 标题模板

优先使用能表达判断的标题：

- “方案 B 更适合当前阶段，但要先控制迁移风险”
- “问题不在流量，而在首屏转化”
- “上线前必须补齐权限、回滚和监控”

只有在文档导航确实需要时，才使用中性的主题标题，如“背景”“附录”“参考资料”。

## 可执行教程

```text
目标：完成后读者将得到什么？
前置条件：版本、权限、账号、依赖和平台要求是什么？
执行上下文：命令在哪个目录、终端或环境中运行？
步骤：按顺序执行什么？
预期结果：关键步骤成功时会看到什么？
最终验证：如何独立确认任务已经完成？
失败恢复：常见错误如何识别和处理？
平台差异：哪些步骤只适用于特定系统或设备？
```
