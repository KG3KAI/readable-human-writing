# 结构与表达审校清单

## 结构

- [ ] 开头直接回答了读者最关心的问题。
- [ ] 核心结论可以压缩成一句话。
- [ ] 一级理由确实支持核心结论，而不是并列堆砌背景。
- [ ] 同层观点处于相近的抽象层级。
- [ ] 每组观点使用了明确的分类规则，并按有意义的顺序排列。
- [ ] 重要重叠已合并，重要遗漏已标注为边界或待确认事项。
- [ ] 事实、判断、假设、建议和未决问题没有混写。
- [ ] 复合文档有一个主框架，局部模板没有在一级结构中平铺拼接。
- [ ] 相邻章节存在明确的时序、因果、递进、对比、拆解、证据或行动关系。
- [ ] 过渡句揭示真实逻辑，没有掩盖章节顺序或论证断裂。

## 表达

- [ ] 每段只有一个主张，首句通常能概括该段。
- [ ] 标题帮助读者定位结论或问题，而不只是重复主题。
- [ ] 句子使用具体主语和动词，避免空泛名词堆叠。
- [ ] 过渡词确实表达逻辑关系，没有为了“像文章”而添加。
- [ ] 没有重复小结、泛化展望、无来源的“业内认为”或模板化开场。
- [ ] 语气与读者、渠道和作者身份匹配。

## 完整性与交付

- [ ] 数字、引用、链接、代码、专有名词和限定条件保持不变。
- [ ] 结论没有超出证据支持范围。
- [ ] 表格只用于比较；列表只用于并列项或步骤。
- [ ] 文档长度符合渠道和读者的阅读预算。
- [ ] 结尾给出明确的限制、决定、负责人、截止时间或下一步（如确有必要）。
- [ ] 正式文档保留了有用的编号、缩进、代码块、表格和导航结构。

## 问答转文档

- [ ] 已确认读者、用途、篇幅和证据边界，或明确采用了默认值。
- [ ] 正文按主题和论证组织，而不是按对话先后拼接。
- [ ] 重复回答、寒暄和失效的中间结论已清理。
- [ ] 已确认事实、推断、分歧和未决问题有清楚边界。
- [ ] 缺失的关键信息被标记为 `待确认`，没有被擅自补全。

## 教程与操作文档

- [ ] 前置条件、版本、权限和执行上下文完整。
- [ ] 命令可以直接复制，且没有与解释文字混在同一代码块中。
- [ ] 每个关键步骤给出预期结果，全文提供独立的成功验证方法。
- [ ] 常见失败说明判断方式和恢复动作。
- [ ] 平台差异被明确区分，没有混写不兼容的步骤。
- [ ] 已验证、实际观察、原因推断、通用建议和待确认事项边界清楚。
